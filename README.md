## About me

I like building systems that connect clean interfaces with real data, hardware, and infrastructure. My work moves between React interfaces, backend APIs, geospatial data, cloud deployment, and sensors on an off road race car.

I am currently studying Computer Engineering at Toronto Metropolitan University. I also work with TMU Baja Racing as Electrical Lead and Web Developer, where I help build the vehicle electronics and the public team website.

## Current focus

| Area | What I am working on |
| --- | --- |
| Software engineering | React, Next.js, TypeScript, FastAPI, REST APIs, and full stack project work |
| Geospatial systems | Crown land discovery, terrain visualization, Mapbox, PostGIS, and map based planning tools |
| Embedded systems | C++, Arduino, telemetry sensors, RPM sensing, gear position sensing, and oil temperature monitoring |
| Delivery | Firebase, GitHub Actions, Docker, AWS EC2, Nginx, and production focused deployment workflows |

## What I care about

I enjoy projects where the constraints are real. Maps need to be accurate. APIs need to stay reliable. Hardware needs to survive the vehicle. Websites need to be clear enough for people to actually use.

That is the kind of work I want to keep doing: practical systems, thoughtful interfaces, and engineering that holds up after the demo.
